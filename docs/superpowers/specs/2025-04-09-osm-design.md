## 1. 项目概述

### 1.1 背景
企业内部大量使用开源软件，但缺乏统一的管理平台。无法有效追踪哪些系统使用了哪些开源组件，存在合规风险（许可证问题）和安全风险（漏洞无法快速定位影响范围）。

### 1.2 目标
构建一个企业级开源软件治理平台，实现：
- 开源软件资产的集中管理
- 使用情况的完整追溯
- 新引入组件的审批管控
- 可视化的统计与分析

### 1.3 目标用户

| 角色 | 职责 | 核心诉求 |
|------|------|----------|
| 开发人员 | 系统开发维护 | 快速查找可用软件、登记使用、申请新软件 |
| 管理人员 | 平台运营、集团领导 | 全局视图、统计报表、审批管控 |

## 2. 功能模块

### 2.1 系统架构管理

#### 2.1.1 业务域管理
- 域的增删改查
- 域基本信息：名称、描述

#### 2.1.2 业务系统管理
- 系统增删改查
- 系统属性：
  - 名称
  - 归属域（关联）
  - 系统简介
  - 系统负责人（多选，人员信息）

#### 2.1.3 应用管理
- 应用增删改查
- 应用属性：
  - 名称
  - 所属系统（关联）
  - 应用描述

**数据关系**: 域(1) → 系统(N) → 应用(N)

---

### 2.2 开源软件库

由管理人员维护的企业开源软件仓库。

#### 2.2.1 软件信息管理
- 软件增删改查
- 软件属性：
  - 名称
  - 技术类型（分类）：数据库、缓存、消息队列、Web框架、工具类等
  - 许可证类型：MIT、Apache-2.0、GPL、BSD等
  - 软件简介
  - 官方文档链接
  - 状态：草稿/已发布/已下线

#### 2.2.2 版本管理
- 多版本支持
- 版本属性：
  - 版本号
  - 版本说明
  - 发布日期
  - 状态：正常/存在漏洞/已弃用

**注意**: MVP阶段不涉及软件包存储（Harbor/Nexus/NAS对接），仅管理元数据。

---

### 2.3 订购与审批流程

开发人员申请使用新开源软件的完整流程。

#### 2.3.1 订购申请

**申请人**: 开发人员

**申请表单字段**:
| 字段 | 必填 | 说明 |
|------|------|------|
| 目标系统 | 是 | 选择已注册的业务系统 |
| 目标软件 | 是 | 选择开源软件库中的软件 |
| 目标版本 | 是 | 选择具体版本 |
| 使用用途 | 是 | 详细描述使用场景和目的 |
| 期望上线时间 | 否 | 计划投入使用的时间 |

**状态流转**:
```
草稿 → 已提交 → 审批中 → 已通过
                ↓          ↓
              已驳回 ← 重新申请
```

**状态说明**:
- **草稿**: 申请人暂存，可随时编辑完善
- **已提交**: 正式提交，等待管理员受理
- **审批中**: 管理员正在审核
- **已通过**: 审批通过，自动创建使用记录（标记"已审批"）
- **已驳回**: 审批未通过，需修改后重新申请

#### 2.3.2 审批处理

**审批人**: 管理人员

**审批操作**:
- 通过：填写审批意见，系统自动创建使用记录
- 驳回：填写驳回原因，申请人可修改后重新提交

---

### 2.4 使用登记

记录业务系统与开源软件的使用关系。

#### 2.4.1 存量补录（历史登记）
- **触发场景**: 系统已在使用某些开源软件，但前期未经过平台审批
- **登记人**: 开发人员
- **登记字段**:
  - 选择系统
  - 选择软件 + 版本
  - 使用用途描述
- **标记类型**: "历史补录"

#### 2.4.2 审批自动记录
- **触发场景**: 订购申请审批通过后自动生成
- **标记类型**: "已审批"

#### 2.4.3 开关控制机制
- **初期**: 开放存量补录功能，快速补齐历史数据
- **后期**: 关闭补录入口，强制要求"先订购审批，后使用"
- **控制方式**: 系统配置开关

#### 2.4.4 使用记录管理
- 查看本系统/本人负责系统的使用清单
- 查看使用记录详情
- 使用记录编辑（管理员可编辑，开发人员仅限自己负责的系统的记录）

---

### 2.5 数据统计与可视化（管理员）

#### 2.5.1 统计面板

**核心指标卡片**:
- 开源软件总数
- 业务系统总数
- 待审批申请数量
- 本月新增使用记录数

**分布统计**:
- 按技术类型分布（饼图/柱状图）
- 按许可证类型分布
- 各业务域系统数量分布

**趋势统计**:
- 近30天订购申请趋势
- 近30天新增使用记录趋势

#### 2.5.2 三级关联图谱

**展示层级**: 域 → 系统 → 软件

**交互功能**:
- 支持放大/缩小
- 支持拖拽平移
- 点击节点查看详情
- 支持筛选（按域、按软件类型等）

**视图模式**:
- 全局视图：展示所有域-系统-软件关系
- 单域视图：聚焦某个业务域及其下属系统
- 单系统视图：展示某个系统的软件使用情况
- 单软件视图：展示某个软件被哪些系统使用

#### 2.5.3 明细报表
- 系统维度：导出某系统的开源软件使用清单
- 软件维度：导出某软件被哪些系统使用
- 域维度：导出某域的所有系统软件使用情况

- 可以看到所有数据
- 拥有所有操作权限

## 6. 非功能需求

### 6.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 后端 | Java 17 + Spring Boot 3.x | 主流企业级框架 |
| 前端 | React 18 + Ant Design 5.x | 丰富的企业级组件 |
| 数据库 | MySQL 8.0 | 关系型数据存储 |
| ORM | MyBatis-Plus | 简化数据访问 |
| 图表 | ECharts / D3.js | 统计图表、关系图谱 |

### 6.2 性能需求

- 页面加载时间 < 2秒
- 查询响应时间 < 500ms
- 支持并发用户数 > 100

### 6.3 安全需求

- 用户身份认证（MVP阶段简单实现，预留SSO扩展接口）
- 数据权限控制（基于角色的数据隔离）
- 操作日志记录（关键操作的审计）

### 6.4 部署需求

- 开发环境：本地启动，简单配置
- 生产环境：Docker容器化，支持K8s部署
- 配置文件外置，支持环境变量注入
