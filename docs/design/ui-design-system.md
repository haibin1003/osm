# OSM 平台 UI 设计规范

> 简洁浅色风格，专业数据可视化，支持下钻查看

---

## 1. 设计理念

### 1.1 设计目标

- **专业感**：适合集团领导查看的高端数据面板
- **清晰性**：信息层次分明，重点数据突出
- **交互性**：支持下钻查看详情，数据可追溯
- **一致性**：统一的设计语言，降低学习成本

### 1.2 设计原则

```
1. 数据优先：让数据本身成为视觉焦点
2. 呼吸感：适当的留白，避免信息拥挤
3. 层次分明：通过色彩、字号、间距建立视觉层级
4. 即时反馈：交互操作有明确的视觉响应
```

#### 图谱实现参考

```tsx
interface GraphNode {
  id: string;
  name: string;
  type: 'domain' | 'system' | 'software';
  level: 1 | 2 | 3;
  data?: any;  // 原始数据，用于下钻
}

interface GraphEdge {
  source: string;
  target: string;
  relation: string;
}

interface RelationGraphProps {
  nodes: GraphNode[];
  edges: GraphEdge[];
  viewMode: 'global' | 'domain' | 'system' | 'software';
  onNodeClick: (node: GraphNode) => void;
  onNodeDoubleClick: (node: GraphNode) => void;  // 下钻
}

// 使用
<RelationGraph
  nodes={graphData.nodes}
  edges={graphData.edges}
  viewMode="global"
  onNodeClick={handleNodeSelect}
  onNodeDoubleClick={(node) => {
    // 下钻到详情页
    if (node.type === 'system') {
      navigate(`/systems/${node.id}/detail`);
    } else if (node.type === 'software') {
      navigate(`/software/${node.id}/usage`);
    }
  }}
/>
```

## 7. 实现建议

### 7.1 推荐组件库

| 组件 | 用途 | 安装 |
|------|------|------|
| Ant Design 5.x | 基础UI组件 | `npm install antd` |
| Ant Design Charts | 统计图表 | `npm install @ant-design/charts` |
| G6 / X6 | 关系图谱 | `npm install @antv/g6` |
| CountUp.js | 数字动画 | `npm install react-countup` |

### 7.2 图表选型

| 场景 | 推荐组件 | 类型 |
|------|----------|------|
| 分布统计 | Pie / Donut | 饼图/环形图 |
| 趋势统计 | Line / Area | 折线/面积图 |
| 对比统计 | Column / Bar | 柱状图 |
| 完成率 | Gauge | 仪表盘 |
| 关系展示 | G6 Graph | 力导向图 |
| 层级展示 | G6 Tree | 树形图 |

### 7.3 性能优化

```
1. 大数据量表格：虚拟滚动
2. 图表：数据采样，避免过多节点
3. 图谱：节点分层加载，先加载1-2级
4. 图片：懒加载
5. 路由：代码分割，按需加载
```

## 8. 设计检查清单

### 视觉检查

```
□ 色彩符合设计系统
□ 间距使用规范值（4/8/16/24/32）
□ 字体层级清晰（主/次/辅助）
□ 卡片有适当的阴影和圆角
□ 图表配色协调且有意义
```

### 交互检查

```
□ 所有可点击元素有 hover 效果
□ 数据加载有 loading 状态
□ 操作成功/失败有反馈提示
□ 面板数据支持下钻查看详情
□ 图谱支持缩放、拖拽、选中
```

### 响应式检查

```
□ 大屏（1920px+）充分利用空间
□ 笔记本（1366px）正常显示
□ 平板（768px）自适应
□ 关键功能在小屏可用
```
